Programming language vs Implementation

programming language
itself blueis only specification
of syntax and semantics
implementation
of programming language
is concrete program, build for specific targets set, with concrete standard (runtime) library set shipped with
reference implementation

some widely known languages (CPython2, Ruby MRI) has some dominatic implementation1.1, known as greenreference implementation: if you write your variant of language system, and want it to be compatible with language you name, you can test it using ref implementation, every test must behave identical.

So reference implementation can be used as language specification includes not only syntax, but semantics too. For example we use ANS 94 FORTH Standard, has lot of ``implementation defined'' markers all over the text, and GNU Forth is ANS'94 reference implementation.
